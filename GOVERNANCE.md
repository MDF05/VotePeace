# Project Governance

This document outlines the governance model for the **VotePeace** project. It describes the roles and responsibilities of community members and the decision-making process.

## Roles

### 1. Maintainer
The Maintainer has final authority over the project's direction.
-   **Current Maintainer**: Muhammad Dava Fahreza (@MDF05)
-   **Responsibilities**:
    -   Merging Pull Requests.
    -   Setting the Roadmap.
    -   Managing Releases.
    -   Enforcing the Code of Conduct.

### 2. Contributors
Contributors are community members who submit code, documentation, or issues.
-   **Responsibilities**:
    -   Submitting quality code.
    -   Following the Contribution Guidelines.
    -   Participating in discussions respectfully.

## Decision Making

### Consensus Seeking
We aim for consensus among contributors. Major changes are discussed in GitHub Issues or Pull Request comments.

### Tie-Breaking
In cases where no consensus can be reached, the Maintainer makes the final decision based on the project's long-term goals.

## Release Process
Releases are managed by the Maintainer. We follow Semantic Versioning (SemVer).
